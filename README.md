## 2. ฟีเจอร์ทั้งหมด

### ฝั่งลูกค้า

- ดูรายการสินค้าแบบ grid พร้อมรูปหมวดหมู่และจำนวนคงเหลือ
- ค้นหาสินค้า (มี debounce), กรองตามหมวดหมู่, เรียงตามราคา, แบ่งหน้า (pagination)
- หน้ารายละเอียดสินค้า: เลือกไซซ์ S/M/L/XL และจำนวน
- **ตะกร้าสินค้า** — เพิ่ม/ลด/ลบสินค้า ยอดรวมอัปเดตทันที ตะกร้าไม่หายตอน refresh
  (สินค้าเดียวกันคนละไซซ์นับเป็นคนละรายการ)
- สั่งซื้อ (Checkout) พร้อมกรอกที่อยู่จัดส่ง
- ดูประวัติออเดอร์ + ยกเลิกออเดอร์ได้ตอนยังไม่ชำระเงิน (ระบบคืนสต็อกอัตโนมัติ)

### ฝั่งผู้ดูแลระบบ (หลังร้าน)

- **Dashboard ยอดขาย** — การ์ดสรุปยอดรวม, กราฟยอดขายรายวัน 7 วันล่าสุด,
  สินค้าขายดี Top 5, จำนวนออเดอร์แยกตามสถานะ (คำนวณด้วย MongoDB Aggregation)
- จัดการสินค้า: เพิ่ม / แก้ไข / ลบ พร้อม**อัปโหลดรูปสินค้าจริง**
  (จำกัดชนิดไฟล์ jpg/png/webp และขนาดไม่เกิน 5MB — ตรวจฝั่ง server)
- ดูออเดอร์ทั้งหมดของทุกคน
- เปลี่ยนสถานะออเดอร์ตามลำดับ: รอชำระเงิน → ชำระแล้ว → กำลังจัดส่ง → สำเร็จ
- ยกเลิกออเดอร์ (คืนสต็อกอัตโนมัติ)

### ระบบแจ้งเตือน (Toast Notification)

- ทุกการกระทำที่ไม่ถูกต้องจะมีแจ้งเตือนมุมขวาบน **ระบุสาเหตุชัดเจน** เช่น
  "สต็อกมี 5 ชิ้น แต่มีในตะกร้าแล้ว 4 ชิ้น", "ไฟล์ใหญ่ 7.2MB — เกินกำหนด 5MB",
  "เปลี่ยนสถานะจาก paid เป็น completed ไม่ได้"
- แยก 3 ระดับ: สำเร็จ (เขียว) / เตือน (เหลือง) / ผิดพลาด (แดง) หายเองใน 5 วินาที
- **Session หมดอายุ** — ถ้า token หมดอายุระหว่างใช้งาน ระบบ logout ให้อัตโนมัติ
  พร้อมแจ้งสาเหตุและพากลับหน้า login (สื่อสารผ่าน custom event จาก API layer)

### ระบบหลังบ้าน

- **ตัดสต็อกแบบ atomic** — กันกรณีสองคนกดซื้อสินค้าชิ้นสุดท้ายพร้อมกัน
- **Rollback สต็อก** — ถ้าสั่งหลายชิ้นแล้วชิ้นใดชิ้นหนึ่งไม่พอ ระบบคืนสต็อกที่ตัดไปแล้วทั้งหมด
- ราคาคิดจากฐานข้อมูลเสมอ ไม่เชื่อราคาที่ client ส่งมา
- Seed สินค้า 13 รายการ + บัญชี admin ให้อัตโนมัติตอนรันครั้งแรก

### 3.1 โฟลเดอร์ `frontend/` — ส่วนแสดงผล (Client)

| โฟลเดอร์/ไฟล์ | หน้าที่ |
|---------------|---------|
| `src/main.jsx` | จุดเริ่มต้นแอป ครอบ `BrowserRouter` + `CartProvider` |
| `src/App.jsx` | Navbar (มี badge จำนวนสินค้าในตะกร้า) + routing ทุกหน้า |
| `src/api.js` | ตัวกลางเรียก API แนบ JWT token อัตโนมัติ + ดักจับ session หมดอายุ |
| `src/context/CartContext.jsx` | state ตะกร้าที่ใช้ร่วมกันทุกหน้า + sync localStorage |
| `src/context/ToastContext.jsx` | ระบบแจ้งเตือนกลาง (สำเร็จ/เตือน/ผิดพลาด พร้อมสาเหตุ) |
| `src/utils.js` | ค่าคงที่และฟังก์ชันที่ใช้ร่วมกัน (หมวดหมู่, สถานะ, format ราคา/วันที่) |
| `src/pages/` | หน้าจอทั้งหมด: หน้าร้าน, รายละเอียดสินค้า, ตะกร้า, checkout, ออเดอร์, login/register, หลังร้าน |
| `src/pages/AdminDashboard.jsx` | Dashboard ยอดขาย: การ์ดสถิติ + กราฟ 7 วัน + Top 5 |
| `nginx.conf` | เสิร์ฟ SPA + ส่งต่อ `/api` ไป backend |

### 3.2 โฟลเดอร์ `backend/` — ส่วนประมวลผล (Server / REST API)

| โฟลเดอร์/ไฟล์ | หน้าที่ |
|---------------|---------|
| `src/server.js` | จุดเริ่มต้นเซิร์ฟเวอร์ + เรียก seed อัตโนมัติ |
| `src/config/db.js` | เชื่อมต่อ MongoDB |
| `src/models/` | Schema: `User`, `Product` (มี stock/sizes), `Order` (เก็บราคา ณ ตอนซื้อ) |
| `src/routes/` | เส้นทาง API: auth, products, orders |
| `src/controllers/` | Logic จริง: ตัดสต็อก atomic + rollback, state machine, aggregation สถิติ |
| `src/middleware/` | เช็ค JWT token, สิทธิ์ admin และตรวจไฟล์อัปโหลด (multer) |
| `src/seed.js` | สินค้า demo 13 รายการ + บัญชี admin (สร้างเฉพาะตอน DB ว่าง) |

## 4. API Endpoints

| Method | Endpoint | สิทธิ์ | หน้าที่ |
|--------|----------|--------|---------|
| POST | `/api/auth/register` | ทุกคน | สมัครสมาชิก |
| POST | `/api/auth/login` | ทุกคน | เข้าสู่ระบบ (รับ JWT token) |
| GET | `/api/products` | ทุกคน | รายการสินค้า (search / category / sort / page) |
| GET | `/api/products/:id` | ทุกคน | รายละเอียดสินค้า |
| POST | `/api/products` | admin | เพิ่มสินค้า |
| PUT | `/api/products/:id` | admin | แก้ไขสินค้า |
| DELETE | `/api/products/:id` | admin | ลบสินค้า |
| POST | `/api/uploads` | admin | อัปโหลดรูปสินค้า (multipart, ตอบ URL กลับ) |
| GET | `/api/uploads/:file` | ทุกคน | เรียกดูรูปที่อัปโหลดแล้ว |
| POST | `/api/orders` | สมาชิก | สั่งซื้อ — ตัดสต็อกแบบ atomic, ไม่พอสั่งไม่ได้ |
| GET | `/api/orders/me` | สมาชิก | ออเดอร์ของตัวเอง |
| GET | `/api/orders` | admin | ออเดอร์ทั้งหมด |
| GET | `/api/orders/stats` | admin | สถิติยอดขาย (aggregation: รายวัน 7 วัน, Top 5, แยกสถานะ) |
| PATCH | `/api/orders/:id/status` | admin | เปลี่ยนสถานะตาม state machine |
| PATCH | `/api/orders/:id/cancel` | เจ้าของออเดอร์ | ยกเลิก (เฉพาะตอนรอชำระ) + คืนสต็อก |

## 5. วิธีรันโปรเจค

### 5.1 แบบ Docker (แนะนำ — คำสั่งเดียวจบ)

> ต้องติดตั้ง [Docker Desktop](https://www.docker.com/products/docker-desktop/) ก่อน

**ขั้นตอนที่ 1** — เปิด Terminal แล้วเข้าโฟลเดอร์โปรเจค

```bash
cd 05-clothing-store
```

**ขั้นตอนที่ 2** — สั่ง build และรันทั้งระบบ

```bash
docker compose up -d --build
```

**ขั้นตอนที่ 3** — เปิดใช้งาน (ระบบ seed สินค้า 13 รายการให้อัตโนมัติ)

| ส่วน | URL |
|------|-----|
| หน้าเว็บ | http://localhost:3001 |
| API | http://localhost:5002 |

**บัญชีสำหรับ Login ทดสอบ**

| บทบาท | อีเมล | รหัสผ่าน | ที่มา | ใช้ทำอะไรได้ |
|--------|-------|----------|-------|--------------|
| Admin | `admin@shop.com` | `admin123` | ระบบ seed ให้อัตโนมัติตอนรันครั้งแรก | เข้าเมนู "หลังร้าน": Dashboard ยอดขาย, จัดการสินค้า/รูป, อัปเดตสถานะออเดอร์ |
| ลูกค้า (User) | `customer@test.com` | `123456` | สมัครผ่านหน้าเว็บ (มีเฉพาะเครื่องที่ทดสอบไว้) | เลือกซื้อ, ตะกร้า, สั่งซื้อ, ดู/ยกเลิกออเดอร์ของตัวเอง |

> ถ้ารันบนเครื่องใหม่ (DB ว่าง) จะมีเฉพาะบัญชี Admin — บัญชีลูกค้ากด "สมัครสมาชิก"
> สร้างเองได้เลย ทุกบัญชีที่สมัครใหม่จะได้ role เป็น user โดยอัตโนมัติ

## 6. ขั้นตอนการใช้งานระบบ

**ขั้นตอนที่ 1 — เลือกซื้อสินค้า**
เปิดหน้าร้าน → ค้นหา/กรองหมวดหมู่/เรียงราคา → กดสินค้าเพื่อดูรายละเอียด

**ขั้นตอนที่ 2 — เพิ่มลงตะกร้า**
เลือกไซซ์ → เลือกจำนวน → กด "เพิ่มลงตะกร้า" (ไอคอนตะกร้าบน navbar จะแสดงจำนวน)

**ขั้นตอนที่ 3 — สั่งซื้อ**
เปิดตะกร้า → ปรับจำนวน/ลบได้ → กด "ดำเนินการสั่งซื้อ"
(ถ้ายังไม่ login ระบบพาไปหน้า login ก่อน) → กรอกที่อยู่จัดส่ง → ยืนยัน

**ขั้นตอนที่ 4 — ติดตามออเดอร์**
หน้า "ออเดอร์ของฉัน" แสดงสถานะปัจจุบัน และยกเลิกได้ตอนยังรอชำระเงิน

**ขั้นตอนที่ 5 — ฝั่งแอดมิน (หลังร้าน)**
Login ด้วยบัญชี admin → เมนู "หลังร้าน" → จัดการสินค้าและอัปเดตสถานะออเดอร์
(รอชำระ → ชำระแล้ว → กำลังจัดส่ง → สำเร็จ)

## 7. จุดเด่นของโปรเจค

> เหมาะสำหรับใช้เล่าตอนสัมภาษณ์งาน

1. **ตัดสต็อกแบบ atomic กัน race condition** — ใช้ `findOneAndUpdate` เงื่อนไข
   `stock >= quantity` พร้อม `$inc` ในคำสั่งเดียว ถ้าสองคนซื้อชิ้นสุดท้ายพร้อมกัน
   จะสำเร็จแค่คนเดียว
2. **Rollback สต็อก** — ออเดอร์หลายรายการ ถ้าชิ้นใดไม่พอ ระบบคืนสต็อกที่ตัดไปแล้วทั้งหมด
   (ทำ compensation แบบ manual แทน transaction)
3. **สถานะออเดอร์เป็น state machine** — เปลี่ยนได้เฉพาะลำดับที่ถูกต้อง
   เช่น ห้ามข้ามจาก "รอชำระ" ไป "สำเร็จ" และการยกเลิกจะคืนสต็อกเสมอ
4. **ราคาจากฐานข้อมูลเสมอ** — order เก็บชื่อ/ราคา ณ ตอนซื้อไว้ในตัวเอง
   ประวัติไม่เพี้ยนแม้สินค้าถูกแก้ราคาทีหลัง และไม่เชื่อราคาที่ client ส่งมา
5. **ตะกร้าด้วย Context API + localStorage** — state ข้ามหน้าโดยไม่ต้องใช้ library เสริม
   และไม่หายตอน refresh
6. **UX รายละเอียดเล็ก ๆ** — debounce การค้นหา, badge จำนวนในตะกร้า, pagination,
   ปุ่ม admin แสดงเฉพาะ transition ที่กดได้จริง
7. **Dashboard ด้วย MongoDB Aggregation Pipeline** — `$match → $group → $sort`
   สรุปยอดรายวัน (จัดกลุ่มตาม timezone ไทย), `$unwind` แตกรายการสินค้าเพื่อหา Top 5
   และนับยอดขายเฉพาะสถานะที่ชำระแล้วเท่านั้น
8. **อัปโหลดไฟล์อย่างปลอดภัย** — multer ตรวจ mimetype + จำกัดขนาด 5MB ฝั่ง server,
   ตั้งชื่อไฟล์ใหม่กันชื่อซ้ำ/ชื่ออันตราย และเก็บใน Docker volume แยก
   (รูปไม่หายแม้ rebuild image)
9. **แจ้งเตือนบอกสาเหตุเสมอ (UX)** — toast กลางทั้งแอปผ่าน Context, ตรวจ input
   ซ้ำสองชั้น (เตือนทันทีฝั่ง client + บังคับจริงฝั่ง server) และจัดการ token
   หมดอายุด้วย custom event `auth:expired` → logout อัตโนมัติพร้อมเหตุผล
10. **Deploy ด้วย Docker** — multi-stage build + nginx reverse proxy เหมือนโปรเจค 04
